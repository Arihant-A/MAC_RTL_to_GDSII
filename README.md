## Project Overview

This project implements a parameterized `n=32` bit MAC unit and walks the complete digital design flow:

```
RTL (Verilog) → Behavioral Sim → Synthesis (Yosys) → STA (OpenSTA)
→ DFT/ATPG (Fault) → Scan Insertion (Manual Mux-Based) → Scan ATPG
```

The design targets the **OSU 0.18µm standard cell library** and is fully reproducible with open-source tools.

## Flow Stages

### 1. RTL Design & Simulation

A parameterized MAC module accumulates `result += a * b` on each enabled clock cycle.

```verilog
always @(posedge clk or posedge rst) begin
    if (rst)       result <= 0;
    else if (clr)  result <= 0;
    else if (en)   result <= result + (a * b);
end
```

Simulated with Icarus Verilog across three test cases (including signed negative operands).

```bash
iverilog mac.v tb_mac.v
vvp a.out
gtkwave mac.vcd
```

---

### 2. Logic Synthesis

Synthesized using **Yosys** targeting the OSU018 standard cell library.

```bash
yosys mac.ys
```

**Synthesis Statistics:**

| Metric | Value |
|--------|-------|
| Total cells | 7,457 |
| Flip-flops (DFFSR) | 65 |
| Chip area | 270,626 µm² |
| Dominant cells | XOR2X1 (1272), NAND2X1 (1920), OAI21X1 (1054) |

Post-synthesis gate-level simulation was also verified with the OSU018 cell models.

---

### 3. Static Timing Analysis (STA)

Performed using **OpenSTA** with a 11 ns clock constraint.

```bash
sta mac_sta.tcl
```

**Timing Results:**

| Clock Period | Setup Slack | Hold Slack | Status |
|---|---|---|---|
| 10 ns | −0.21 ns (WNS) | 0.37 ns | ❌ Setup Violation |
| 11 ns | +0.79 ns | 0.37 ns | ✅ Met |

**Maximum Operating Frequency:**
```
T_min = 11 ns − 0.79 ns = 10.21 ns  →  F_max ≈ 97.94 MHz
```

The generated `mac.sdf` is back-annotated into the testbench for gate-level timing simulation.

### 4. DFT & ATPG (Non-Scan)

**Key challenge:** Asynchronous reset caused X-propagation in ATPG, dropping coverage to 0%.  
**Fix:** Refactored RTL to use **synchronous reset**, making all flip-flops fully controllable.

```bash
fault synth --liberty osu018_stdcells.lib --top mac -o mac_synth.v mac.v
fault cut --clock clk --reset rst -o mac_cut.v mac_synth.v
fault atpg --cell-model osu018_stdcells.v --clock clk --reset rst \
           --bypassing en=1 --bypassing clr=0 -o mac_atpg.json mac_cut.v
```

**ATPG Results (Non-Scan):**

| Metric | Value |
|--------|-------|
| Fault sites | 22,523 |
| Gates | ~7,005 |
| Fault coverage | **98.45%** |
| Raw test vectors | 100 |
| Compacted vectors | 61 |
| Compaction ratio | 39% |

---

### 5. Scan Chain Insertion & Scan ATPG

**Architecture:** 5 parallel scan chains × 13 flip-flops = 65 FFs total.  
**Method:** Manual mux-based insertion into `synth_mac.v` (avoided automated boundary scan overhead).

Each FF's data input is replaced:
```verilog
// Before
.D(original_data)

// After (scan mux)
.D(shift ? scan_in : original_data)
```

New scan ports added: `shift`, `sin[4:0]`, `sout[4:0]`

**Scan Verification:** Data shifted into `sin[i]` emerges at `sout[i]` exactly **13 clock cycles** later — confirming correct chain construction.

```bash
iverilog -o sim tb_mac_scan.v mac_scan_manual.v osu018_stdcells.v
vvp sim
gtkwave scan.vcd
```

**Scan ATPG Results:**

| Metric | Value |
|--------|-------|
| Fault coverage | **96.50%** |
| Raw test vectors | 100 |
| Compacted vectors | 19 |
| Compaction ratio | **81%** |

## Key Results

| Stage | Result |
|-------|--------|
| Synthesis | 7,457 cells, 270,626 µm² area |
| Max frequency | ~97.94 MHz |
| Non-scan fault coverage | **98.45%** (61 vectors) |
| Scan fault coverage | **96.50%** (19 vectors) |
| Scan chain depth | 13 FFs/chain × 5 chains |

- **Note:** Scan fault coverage (96.50%) is lower than non-scan (98.45%) only because the `--bypassing en=1 --bypassing clr=0` bypass flags used in the non-scan run weren't re-applied for scan ATPG — not because scan hurt testability but the scan run did not re-apply that bypass once `shift`/`sin`/`sout` were exposed as ATPG-controlled ports. With the same bypass constraints applied, scan ATPG would be expected to match or exceed the non-scan result, since scan chains give the ATPG engine direct control over every flip-flop independent of `en`/`clr`.

## How to Reproduce

> Assumes Icarus Verilog, GTKWave, Yosys, OpenSTA, and Fault are installed, and the OSU018 PDK is available.

**1. RTL Simulation**
```bash
cd rtl
iverilog mac.v tb_mac.v && vvp a.out
```

**2. Synthesis**
```bash
cd synthesis
yosys mac.ys
```

**3. STA**
```bash
cd sta
sta mac_sta.tcl
```

**4. Non-Scan ATPG**
```bash
cd dft
fault synth --liberty <path>/osu018_stdcells.lib --top mac -o mac_synth.v ../rtl/mac_sync.v
fault cut --clock clk --reset rst -o mac_cut.v mac_synth.v
fault atpg --cell-model <path>/osu018_stdcells.v --clock clk --reset rst \
           --bypassing en=1 --bypassing clr=0 -o mac_atpg.json mac_cut.v
```

**5. Scan ATPG**
```bash
cd dft/scan
fault cut --clock clk --reset rst -o mac_scan_cut.v mac_scan_manual.v
fault atpg --clock clk --cell-model <path>/osu018_stdcells.v \
           -o mac_scan_atpg.json mac_scan_cut.v
```
